# Module quality standard

A module meets platform standard only if it satisfies all of the following:

## 1. source fidelity
The module reflects actual course materials accurately.

## 2. pedagogical depth
The module reaches at least mikro1-level clarity, structure, and training quality.

## 3. active recall support
The module contains attempt-first learning paths, not just explanations.

## 4. interactive reasoning
Where appropriate, the module preserves or adds graphs, simulations, computational tools, or other subject-appropriate active learning tools.

## 5. assessment readiness
The module can support drills, quick tests, exams, mistake tracking, and future adaptive review.

## 6. consistency
The module fits the platform backbone without losing subject-specific identity.
