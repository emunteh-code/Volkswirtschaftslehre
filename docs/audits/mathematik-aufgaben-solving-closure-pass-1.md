# Mathematik Aufgaben Solving Closure Pass 1

## Scope
Correct the `mathematik` Aufgaben tab so it trains for the actual exam format: solving, not mainly theory recall.

## Benchmark reference
- `mikro1` Aufgaben tab: dense, exam-facing, no generic filler cards pretending to be useful
- actual `Mathematik` Kleinübungen in `source-materials/Mathematik/Mathematik/Kleinübung/`

## Reproduced issue before fix
1. `mathematik` concept pages used the shared `buildSupplementalPracticeTasks(...)` fallback, which padded the guided tasks with theory/intuition/meta prompts once the authored task count was below the shared floor.
2. `Prüfungstransfer` on `mathematik` pages was generated by the generic theory-heavy exam-drill builder (`Kernidee`, `Theorieblock`, `Klausurmuster`, etc.).
3. The visible result was mathematically serious but still too recall-oriented for a module whose exam style is dominated by solving routes from the Kleinübungen.

## Implementation plan
1. Keep the shared renderer architecture, but add opt-in hooks so one module can avoid generic filler without affecting the others.
2. Make `mathematik` stop auto-padding guided tasks with generic theory prompts.
3. Make `mathematik` build `Prüfungstransfer` from its real solving tasks and warning/trap structure instead of the generic recall-style drill generator.
4. Preserve the current authored math tasks and use their Kleinübung lineage as the visible exam-transfer basis.

## Success criteria
- `Geführte Aufgaben` in `mathematik` no longer get generic recall filler.
- `Prüfungstransfer` in `mathematik` is solving-focused and visibly built around actual exercise families.
- Other modules keep their existing behavior unless explicitly opted in later.

## Visible improvements made
1. The shared renderer now supports two safe opt-in hooks:
   - `minimumPracticeTasks`
   - `examDrillsById`
2. `mathematik` opts out of the generic guided-task filler by setting `minimumPracticeTasks: 0`.
3. `mathematik` now uses a module-local `examDrillsById` map built from the real math task layer, so `Prüfungstransfer` is solving-first instead of theory-first.
4. Each math transfer deck now uses:
   - solving questions from the concept’s task family
   - full stepwise solution logic
   - a final error-control card tied to the concept’s typical trap

## Remaining honest gap
- This pass closes the visible recall-vs-solving mismatch on the Aufgaben tab.
- It does not yet make every single math task a verbatim Kleinübung import. Many are still `source-distilled` rather than direct copies, even though the transfer logic is now correctly solving-first.
- A later pass could still deepen density by adding more direct-source guided tasks per concept, especially in `Summen/Logik`, `Analysis II` and the later optimization blocks.
