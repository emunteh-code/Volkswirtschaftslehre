# Learning data model

## required entities
- module
- chapter
- subtopic
- concept
- subskill
- formula
- graphFamily
- sourceReference
- learningObject
- drill
- question
- examTask
- attempt
- mistakeLogEntry
- masteryState
- reviewItem

## design principles
1. support source fidelity
2. support module-specific interactivity
3. support exams, quick tests, and mixed review
4. support future spaced repetition
5. support dashboard metrics based on real performance
